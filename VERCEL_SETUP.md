# Инструкция по настройке Vercel

## Шаг 1: Установка Vercel CLI (опционально)

Если вы хотите деплоить через командную строку:

```bash
npm install -g vercel
```

## Шаг 2: Подключение проекта к Vercel

### Вариант A: Через веб-интерфейс (рекомендуется)

1. Зайдите на [vercel.com](https://vercel.com) и войдите в аккаунт (можно через GitHub)
2. Нажмите "Add New Project"
3. Импортируйте ваш репозиторий GitHub (или загрузите файлы)
4. Vercel автоматически определит настройки проекта
5. Нажмите "Deploy"

### Вариант B: Через командную строку

```bash
# В корне проекта
vercel

# Следуйте инструкциям:
# - Войдите в аккаунт (если нужно)
# - Подтвердите настройки проекта
# - Дождитесь деплоя
```

## Шаг 3: Настройка переменных окружения

Для работы API функции отправки отзывов в Telegram нужно настроить переменные окружения в Vercel:

1. **Получите токен бота:**

   - Напишите [@BotFather](https://t.me/BotFather) в Telegram
   - Отправьте команду `/newbot`
   - Следуйте инструкциям и получите токен
   - Пример токена: `123456789:ABCdefGHIjklMNOpqrsTUVwxyz`

2. **Получите Chat ID:**

   - Напишите [@userinfobot](https://t.me/userinfobot) в Telegram
   - Бот покажет ваш Chat ID
   - Или создайте группу и добавьте туда [@RawDataBot](https://t.me/RawDataBot)
   - Пример Chat ID: `123456789`

3. **Настройте переменные окружения в Vercel:**
   - В панели Vercel перейдите в настройки проекта (Settings)
   - Откройте раздел "Environment Variables"
   - Добавьте следующие переменные:
     - **TELEGRAM_BOT_TOKEN** - вставьте ваш токен бота
     - **TELEGRAM_CHAT_ID** - вставьте ваш Chat ID
   - После добавления переменных перезапустите деплой (Redeploy)

✅ **Преимущество:** Токен бота хранится безопасно на сервере и не виден в исходном коде страницы!

## Шаг 4: Структура проекта

Проект должен иметь следующую структуру:

```
gavhar-taplink/
├── api/
│   └── send-feedback.js    # Serverless функция для отправки отзывов
├── img/                     # Изображения
├── index.html              # Главная страница
├── script.js               # JavaScript код
├── style.css               # Стили
└── vercel.json             # Конфигурация Vercel
```

## Шаг 5: Проверка работы

После деплоя:

1. Откройте ваш сайт (URL будет предоставлен Vercel)
2. Проверьте работу формы "ОТЗЫВЫ И ПРЕДЛОЖЕНИЯ"
3. Отправьте тестовое сообщение
4. Проверьте, что сообщение пришло в Telegram

## Автоматический деплой

Vercel автоматически деплоит проект при каждом push в основную ветку (main/master) вашего репозитория GitHub.

```bash

# Просмотр логов
vercel logs

# Просмотр информации о проекте

```

## Решение проблем

### API функция не работает

- Проверьте, что переменные окружения установлены правильно в Vercel
- Убедитесь, что TELEGRAM_BOT_TOKEN и TELEGRAM_CHAT_ID заполнены
- Проверьте логи в панели Vercel (Functions → Logs)
- Убедитесь, что после добавления переменных вы перезапустили деплой (Redeploy)
- Проверьте токен бота - он должен быть в формате `число:буквы_и_цифры`
- Убедитесь, что бот активен (напишите ему в Telegram)

### Статические файлы не загружаются

- Убедитесь, что все файлы находятся в корне проекта
- Проверьте пути к файлам в HTML (они должны быть относительными)

### Ошибки деплоя

- Проверьте синтаксис vercel.json
- Убедитесь, что все зависимости установлены (если есть package.json)
